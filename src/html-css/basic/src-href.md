# src 和 href 的区别

> 在 `HTML` 中，‌`src` 和 `href` 都用于指定资源路径，但核心区别在于 `src` 用于嵌入资源并阻塞文档解析，而 `href` 用于建立链接并支持并行加载

## `src` 

- 是 “source”（源）的缩写

- 将外部资源（如图片、脚本、视频）直接嵌入到当前文档中，资源是页面渲染的必需部分

- 同步加载。浏览器解析到 `src` 属性时，会暂停文档处理，优先下载并执行资源（如 `script` 加载JS），完成后才继续渲染

- 直接影响页面布局和渲染。例如，`img` 的src缺失会导致图片占位符显示，`script` 的 `src` 错误可能中断脚本执行。‌

- 常用在 `img`、`script`、`iframe`、`audio`、`video` 等标签

## `href`

- 是 “Hypertext Reference”（超文本引用）的缩写

- 建立当前元素与外部资源（如CSS样式表、超链接目标）的关联，资源是可选引用，不会直接嵌入文档

- 异步加载，浏览器识别资源后（如CSS文件），会并行下载，不阻塞文档解析

- 常用在 `a`、`link` 标签
